# Дизайн: Назначение модулей методистам админом

## Дата: 2026-05-31

---

## 1. Цель

Админ может назначать published-модули методистам для обучения/ознакомления. Назначенный методист видит модуль в списке и может просматривать его содержимое (уроки, эвристики), но **не может** редактировать, удалять, менять статус или назначать других.

## 2. Модель данных

**Никаких изменений.** Используем существующую таблицу `module_assignments` (user_id теперь может быть seminarist, candidate **или** methodist).

## 3. Backend изменения

### 3.1 `_can_access_module`

Для methodist:
- Свои модули (author_id == current_user.id) — доступ всегда
- Назначенные published модули — доступ для просмотра
- Назначенные draft/archived — недоступны (API не позволяет назначить, но защита на всякий случай)

### 3.2 `list_modules` для methodist

Нужно вернуть свои модули (author) **И** назначенные published модули. Добавляем новую CRUD-функцию `get_multi_for_methodist` с `OR` условием:
```sql
WHERE (author_id = :author_id OR id IN :assigned_ids)
  AND (status = :status если задан)
```

### 3.3 `_can_modify_module`

**Без изменений.** Methodist может модифицировать только свои модули (author_id == current_user.id).

### 3.4 Assignment endpoints

**Без изменений.** `POST /modules/{id}/assignments` уже доступен admin'у. Admin может назначить любой published модуль любому пользователю, включая methodist.

## 4. Frontend изменения

### 4.1 `ModuleDetailPage.tsx`

Добавить проверку `isOwnModule = module?.author_id === user?.id`.

Кнопки управления (изменение статуса, редактирование, удаление, создание теста, добавление урока/эвристики, секция назначений) показываются только если:
- admin ИЛИ
- methodist + isOwnModule

### 4.2 Визуальное различие

Если методист открыл **назначенный** модуль (не свой), рядом со статусом добавляется бейдж «Назначен», чтобы пользователь понимал read-only режим.

## 5. Тесты

- Методист видит свои модули + назначенный published в `list_modules`
- Методист 200 на GET назначенного published модуля
- Методист 403 на PATCH назначенного модуля
- Методист 403 на DELETE назначенного модуля
- Методист может просматривать уроки и эвристики назначенного модуля
- Админ может назначить published модуль методисту

## 6. Коммиты

1. `content: методист может просматривать назначенные модули`
2. `content: тесты для назначения модулей методистам`
3. `frontend: read-only режим для назначенных модулей методиста`
4. `fix: lint и форматирование`
